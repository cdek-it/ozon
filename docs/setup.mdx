# Настройка интеграции

Настройка интеграции доступна после передачи токена доступа Ozon в личном кабинете СДЭК.
Подробнее об этом можно прочитать на [отдельной странице](./ozon-key.mdx).

<Image img={require('./img/setup/init.png')} />

:::danger

Если вы **удаляете интеграцию** после настройки, или **отзываете токен доступа в личном кабинете Ozon**, или
**изменяете в личном кабинете СДЭК токен на некорректный**, то тем самым вы даете согласие на удаление всех данных, связанных с интеграцией, в том числе, и созданных накладных.

Если при этом часть накладных будут в пути, то они будут возвращены отправителю **с выставлением счета на оплату за
полную стоимость доставки**.

:::


### Данные о магазине {#shop}

Основные настройки задаются в разделе настроек магазина.

<details>
    <summary>Секция данных о магазине</summary>

    <Image img={require('./img/setup/shop/main2.png')} />

    Данные из поля названия будут использоваться в качестве продавца в накладной.

    <Image img={require('./img/autopack.png')} />

    По умолчанию новые заказы автоматически будут переводиться интеграцией в статус Ожидают отгрузки.  
    Для отключения этой функциональности можно нажать Выключить автосборку. В этом случае если у покупателя было несколько заказов их можно будет разбить в ЛК Озона.  [Инструкция как разбить заказ на несколько отправлений](https://seller-edu.ozon.ru/mobile-app/rabota-s-zakazamy).


</details>

<details>
    <summary>Секция тарифов</summary>

    <Image img={require('./img/setup/shop/tariff.png')} />

    Данные из полей секции будут участвовать в алгоритме определения тарифа, с которым будет создана накладная.

    :::warning

    При выборе тарифов рекомендуется выбирать как можно больше вариантов, включая тариф `DBS` и `Экспресс`.
    Это позволит сократить количество необработанных накладных.

    :::

    Подробнее про алгоритм подбора тарифов можно прочитать в [отдельной статье](./creation.mdx#create).

    <details>
        <summary>Нельзя отключить `Дополнительный сбор за объявленную стоимость`</summary>

        Данная услуга является обязательной для Интернет-Магазинов, поэтому отключить ее нельзя.
        Стоимость товара подтягивается из карточки товара Ozon. Дополнительный сбор рассчитывается
        автоматически на основе стоимости товара.

        Подробнее можно прочитать в [Регламенте возмездного оказания курьерских услуг](https://www.cdek.ru/ru/oferta/).

    </details>

</details>

<details>
    <summary>Секция товаров</summary>

    <Image img={require('./img/setup/shop/goods.png')} />

    :::info

    Данная секция не является обязательной к заполнению.

    :::

    Если у карточки товара в Ozon не указан параметр, то будет использоваться значение из поля в данной
    секции для вычисления стоимости доставки.

</details>

<details>
    <summary>Секция уведомлений</summary>

    <Image img={require('./img/setup/shop/notifications.png')} />

    :::info

    Данная секция не является обязательной к заполнению.

    :::

    Настройки в данной секции позволяют оперативно получать уведомления, связанные с интеграцией.

    - Об ошибках: каждый раз при попытке создания накладной, если возникла ошибка, будет отправлено сообщение
    - О заказах: в соответствии с выбранными настройками, будет отправлен список обработанных заказов и созданных к
    ним накладных
    - Новости: при добавлении новых функций или изменении работы интеграции, будет отправлено сообщение с описанием изменений

    :::note

    В настоящий момент работает только рассылка уведомлений об ошибках на почту.

    :::
</details>

:::note

После нажатия на кнопку **Сохранить**, настройки магазина будут применены в течение 5 минут.

:::

### Данные о складе

Для каждого склада задаются условия отправки груза и данные для накладной.

Если вы не создавали склады в Ozon, то сделайте это по [инструкции](https://seller-edu.ozon.ru/rfbs/logistics-settings).

:::warning

Скорость доставки в личном кабинете Ozon нужно указывать для realFBS - Standard.

<Image img={require('./img/setup/warehouse/type1.png')} />

В новом методе доставки выбрать доставку через CДЭК.
Затем выбрать вариант `Уже есть договор`.

<Image img={require('./img/setup/warehouse/contract1.png')} />

:::tip

При выборе тарифа, есть возможность выбрать доставку до ПВЗ или курьером.

:::note

При выборе в личном кабинете СДЭК тарифа, отличного от выбранного на Ozon, заказ будет создан с тарифом, указанным в
личном кабинете СДЭК.

:::


:::note

Склады и их наименование загружаются из Ozon автоматически раз в сутки.

Для обновления данных вручную, необходимо нажать на кнопку **Синхронизировать склады**.

:::warning

Процесс ручной синхронизации может занимать продолжительное время в зависимости от количества складов и текущей
нагрузки на систему.

:::

<details>
    <summary>Основная секция</summary>

    <Image img={require('./img/setup/warehouse/main.png')} />

  В данной секции можно:
- **Активировать/деактивировать интеграцию склада**
- **Задать город**, в котором находится склад

**Важно:** Для включения управления складом необходимо **включить радиобатон "Синхронизировать склады"**. 

    :::note

    В настоящий момент вызов курьера на ваш склад через данную интеграцию недоступен.

    :::

    :::info

    Заказы, созданные покупателями для деактивированного склада, не будут загружаться из Ozon.

    :::

</details>

<details>
    <summary>Секция данных в накладной</summary>

    <Image img={require('./img/setup/warehouse/info.png')} />

    Данные из полей будут подставляться в блок отправителя в накладную.

    Обязательными к заполнению являются поля `ФИО`, `Телефон` и `Email`.

</details>

:::note

После нажатия на кнопку **Сохранить**, настройки склада будут применены в течение 5 минут.

:::

### Активация магазина

Для активации магазина, необходимо нажать на кнопку **Активировать**.

<Image img={require('./img/setup/activate.png')} />

:::warning

Активация интеграции станет доступна только после полной настройки магазина и хотя-бы одного склада.

:::

:::danger

**С момента активации магазина должно пройти не менее 30 минут для корректной работы интеграции.**
Это связано с особенностями работы API Ozon.

Любые действия со сборочными заданиями до этого времени могут привести к некорректной работе интеграции и накладные по
таким заданиям созданы не будут.

:::

<details>
    <summary>Деактивация магазина</summary>

    Для деактивации магазина, необходимо нажать на кнопку **Деактивировать**.

    <Image img={require('./img/setup/deactivate.png')} />

    :::note

    Магазин деактивируется в течение 5 минут.

    :::

</details>

После активация магазина, интеграция начнет загружать заказы из Ozon и можно будет работать с
накладными.
